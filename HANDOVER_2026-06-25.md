ボランティアの電話受付を支援するシステムを**本番公開**まで完成させた。

### 公開URL（スマホから即アクセス可）
```
https://tsurumai-system.pages.dev/
```

### 久山くん（ボランティア）の実際の操作（1件あたり2タップ）
1. 電話に出る（スマホ1・スピーカー）
2. 巨大赤ボタン「録音スタート」タップ → AIが会話聞きながら草案作成
3. お客様と話す
4. 電話を切る
5. 「✅承認」タップ → 自動でDB保存＋ドライバー通知＋クリア

### システム構成
```
[スマホ2（操作用）] → https://tsurumai-system.pages.dev/
   ↓
[Cloudflare Pages]（GitHub pushで自動デプロイ）
   ↓ ①データ保存
[Supabase DB] — 場所50/利用者94/ドライバー1/予約895件
   ↓ ②承認時通知
[Supabase Edge Function: notify-driver]（メール送信は次フェーズ）
```

## 🧠 重要なコンテキスト（KTの哲学・設計思想）

### 1. 「AIは構築時のみ・運用には入れない」
> KTの名言：「運営にAIを入れるとぐちゃぐちゃになる」
- AIは設計・構築・一次受付のみ。最終判断は必ず人（ボランティアの承認ボタン）。
- 運用フェーズでAIが勝手に判断しない設計（コパイロット方式）。

### 2. 「現場目線」最優先
> KTの指摘：「電話取ってアプリ切り替えて録音ボタン押して…は絶対現場で使わない」
- 技術者目線でなく、久山くん（ボランティア）が実際にどう動くかを逆算。
- 最終仕様: 「録音ボタン1回＋承認ボタン1回」の2タップ最小構成。

### 3. 「シンプルに始める」
> KTの判断：「最初はできるだけシンプルにスタート。難易度も簡単に」
- VAD自動検出等の高度機能は後回し。まず「動く形」を持つ。

### 4. 「継続ブラッシュアップ」が価値
- 市長面会までダッシュボード等を継続改善中＝KTの本気度の証明。
- AI活用の継続実績（需要予測・音声分析・PoC基盤）を可視化。

### 5. 政治的配慮（必ず守る）
- **しなと議員（信人浩一）**: 市長面会（口頭）でのみ自然に伝達可。**公文書（RFI等）には出さない**。
- **NAIST名**: RFI等の公文書では出さない（フライング防止）。
- **鶴舞地区のみ**: 津久米地区ではない（KT確認済）。
